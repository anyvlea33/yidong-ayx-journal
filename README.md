# yidong-ayx-journal

## 项目简介

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。页面内容涵盖日常记录、笔记整理、静态展示等场景，不针对任何特定域名或网站。所有 HTML 文件均为独立页面，可直接通过浏览器打开查看。

## 目录说明

```
yidong-ayx-journal/
├── index.html          # 仓库首页或页面导航入口
├── pages/              # 存放各独立 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 静态资源文件（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **index.html**：可根据需要作为页面列表或导航入口。
- **pages/**：所有独立 HTML 页面均放置在此目录下，文件名应具有描述性。
- **assets/**：存放页面引用的外部资源，按类型分目录管理。

## 页面归档说明

- 每个 HTML 页面均为自包含的独立文档，可单独部署或浏览。
- 页面内容不依赖外部服务或特定域名，尽量保持离线可访问。
- 归档的页面可能包含实验性内容、临时记录或完整展示，无统一主题限制。
- 建议在页面头部添加简单的标题和说明，便于查阅。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面或改进现有页面。
- 提交新页面时请放置在 `pages/` 目录下，并确保文件名无特殊字符。
- 若页面引用 assets 目录中的资源，请保持路径相对性（如 `../assets/css/style.css`）。
- 请勿在页面中嵌入恶意脚本、跟踪代码或违反开源社区规范的内容。
- 本仓库不承诺对页面内容进行实时审查，但会定期清理无关或失效页面。
